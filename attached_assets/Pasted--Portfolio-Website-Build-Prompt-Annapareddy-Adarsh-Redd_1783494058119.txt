## 4. ANIMATION GUIDELINES (high-polish, still restrained)

Use Framer Motion for section/element choreography, GSAP+ScrollTrigger only for the timeline reveal and chart draw-in so the two libraries don't fight over the same elements.

- Page load: brief branded loading screen, then hero fade/slide-up
- Scroll reveals: fade-up + slight y-offset on section entry, staggered by ~80–120ms per card in a grid
- Navbar: blur/opacity transition on scroll, animated underline/pill for active section
- Stat counters: CountUp triggered on viewport entry, run once
- Timeline: progressive line-draw + node reveal on scroll (GSAP ScrollTrigger)
- Chart: animated line draw-in with eased point reveal
- Buttons: subtle hover elevation/scale (1.02–1.03), no exaggerated bounce
- Optional: magnetic hover only on primary CTA buttons, not every element
- Respect `prefers-reduced-motion` — provide a static fallback

Avoid: parallax overload, animated cursors that obscure content, more than one continuously-looping background animation, motion on body text.
